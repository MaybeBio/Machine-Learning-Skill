# Chapter 15: 规则学习 / Rule Learning

## 核心概念 (Core Idea)

规则学习是符号主义学习的代表，从训练数据中学出一组形如"若...则..."的逻辑规则，具有极强的可解释性和领域知识引入能力。命题规则学习处理"属性-值"表示，一阶规则学习（如FOIL、ILP）可处理对象间关系和递归概念。(西瓜书/南瓜书)

## 融合框架 (Integrated Frameworks)

规则学习的技术体系：

| 层级 | 表达力 | 代表算法 | 搜索策略 | 来源 |
|------|--------|---------|---------|------|
| 命题规则 | 属性-值 | PRISM, CN2, RIPPER | 自顶向下 (特化) | 西瓜书15.1-15.3 |
| 一阶规则 | 关系+变量 | FOIL | 自顶向下 + 变量替换 | 西瓜书15.4 |
| 归纳逻辑程序设计 | 关系+函数+嵌套 | PROGOL, ALEPH | 自底向上 (泛化, 逆归结) | 西瓜书15.5 |

核心策略：序贯覆盖 (Sequential Covering) = 分治 (Separate-and-Conquer)，即逐条学习规则，每学一条就移去被覆盖的样例。(西瓜书)

## 理论推导 (Theoretical Derivation)

### 1. 规则的形式定义

一条规则形如：(西瓜书)

$$
\oplus \leftarrow \mathbf{f}_1 \land \mathbf{f}_2 \land \cdots \land \mathbf{f}_L \tag{15.1}
$$

其中：
- "$\leftarrow$" 左边是**规则头 (head)**，表示判定结论（如"好瓜"）
- "$\leftarrow$" 右边是**规则体 (body)**，为逻辑文字 $\mathbf{f}_k$ 的合取式
- $L$ 为规则长度
- 每个文字 $\mathbf{f}_k$ 是形如 $R(\text{属性}_i, \text{属性值}_{i,j})$ 的布尔表达式

**命题规则 vs 一阶规则**（西瓜书）：
- 命题规则：文字为原子命题，如 `(色泽=青绿)`
- 一阶规则：文字为原子公式，如 `根蒂更蜷(X, Y)`，引入逻辑变量和量词

### 2. 序贯覆盖的形式化

给定正例集 $D^+$ 和反例集 $D^-$，候选文字集 $\mathcal{F} = \{f_k\}$。学习目标：生成能覆盖最多正例、最少反例的规则集。(西瓜书)

规则搜索策略对比 (西瓜书)：

| 策略 | 方向 | 过程 | 适用场景 |
|------|------|------|---------|
| 自顶向下 (Top-down) | 一般→特殊 | 从空规则开始逐次添加文字（特化） | 命题规则学习（泛化性能好、抗噪强） |
| 自底向上 (Bottom-up) | 特殊→一般 | 从具体实例开始逐次删除文字（泛化） | 一阶规则学习（样本少时适用） |

### 3. 剪枝理论

**CN2的似然率统计量 (LRS)**：(西瓜书)

$$
\text{LRS} = 2 \cdot \left( \hat{m}_+ \log_2 \frac{\frac{\hat{m}_+}{\hat{m}_+ + \hat{m}_-}}{\frac{m_+}{m_+ + m_-}} + \hat{m}_- \log_2 \frac{\frac{\hat{m}_-}{\hat{m}_+ + \hat{m}_-}}{\frac{m_-}{m_+ + m_-}} \right) \tag{15.2}
$$

其中 $m_+, m_-$ 为训练集正反例数，$\hat{m}_+, \hat{m}_-$ 为规则覆盖的正反例数。LRS度量规则覆盖分布与经验分布的差异——LRS越大，规则越不可能是偶然。(南瓜书补充)

**FOIL增益**：(西瓜书)

$$
\text{F-Gain} = \hat{m}_+ \times \left( \log_2 \frac{\hat{m}_+}{\hat{m}_+ + \hat{m}_-} - \log_2 \frac{m_+}{m_+ + m_-} \right) \tag{15.3}
$$

与信息增益不同，FOIL增益仅考虑正例信息量，并以覆盖正例数为权重——因为关系数据中正例远少于反例。(南瓜书补充)

### 4. 逆归结的数学基础

**归结原理 (命题逻辑)**：对 $C_1 = A \lor L$ 和 $C_2 = B \lor \neg L$，归结得 $C = A \lor B$。(西瓜书)

定义析合范式删除操作 $(A \lor B) - \{B\} = A$ (15.6)，归结过程：

$$
C = (C_1 - \{L\}) \cup (C_2 - \{\neg L\}) \tag{15.7}
$$

简记为 $C = C_1 \cdot C_2$ (15.8)。

**逆归结**：已知 $C$ 和 $C_1$，求 $C_2$：

$$
C_2 = (C - (C_1 - \{L\})) \lor \{\neg L\} \tag{15.9}
$$

南瓜书推导：从 $C = (C_1 - \{L\}) \lor (C_2 - \{\neg L\})$ 出发，两边删除 $(C_1 - \{L\})$ 再加回 $\{\neg L\}$ 即得。

**一阶逆归结**：利用合一置换 $\theta_1, \theta_2$：(西瓜书)

$$
C_2 = (C - (C_1 - \{L_1\}) \theta_1 \lor \{\neg L_1 \theta_1\}) \theta_2^{-1} \tag{15.16}
$$

四种逆归结操作（西瓜书）：
- **吸收 (Absorption)** (15.10)：$\frac{p \leftarrow A \land B \quad q \leftarrow A}{p \leftarrow q \land B \quad q \leftarrow A}$ — 用新文字替换子句
- **辨识 (Identification)** (15.11)：$\frac{p \leftarrow A \land B \quad p \leftarrow A \land q}{q \leftarrow B \quad p \leftarrow A \land q}$ — 提取共同子句定义新文字
- **内构 (Intra-construction)** (15.12)：从同一文字的规则中提取差异部分为新文字
- **互构 (Inter-construction)** (15.13)：从不同文字的规则中提取共同子句定义新文字

### 5. 最小一般泛化 (LGG)

LGG是能特化两条一阶公式的最特殊公共泛化：(西瓜书)

算法：找出涉及相同谓词的文字，对每个位置较两个常量——相同则保留 $LGG(t,t)=t$，不同则替换为新变量 $LGG(s,t)=V$，并将替换应用于所有出现位置。

RLGG (Relative LGG) 扩展：将样例 $e$ 的初始规则定义为 $e \leftarrow K$，其中 $K$ 为所有背景知识的合取。

## 核心概念 (Key Concepts)

- **覆盖 (Cover)**: 满足规则体条件的样本称为被规则覆盖
- **冲突消解 (Conflict Resolution)**: 多条规则判别不一致时的解决策略（投票法、排序法、元规则法）
- **默认规则 (Default Rule)**: 处理未被任何规则覆盖的样本
- **序贯覆盖 (Sequential Covering)**: 逐条学规则、逐步移去覆盖样例的分治策略
- **集束搜索 (Beam Search)**: 每轮保留最优 $b$ 个候选，缓解贪心算法的局部最优
- **减错剪枝 (REP)**: 用验证集评估，穷举删除操作，保留最优规则集
- **IREP**: REP的增量版，每生成一条规则即剪枝，复杂度从 $O(m^4)$ 降至 $O(m \log^2 m)$
- **合一 (Unification)**: 通过变量置换使两个逻辑表达式相等
- **最一般合一化子 (MGU)**: 最"经济"的合一置换，其他合一化子均可由其复合得到
- **归结商 (Resolution Quotient)**: $C_1 = C/C_2$，逆归结的目标

## 算法步骤 (Algorithm Steps)

### RIPPER算法 (图15.2)
1. $R = \text{IREP}^*(D)$ — 用改进IREP生成初始规则集
2. 对 $i = 0, \ldots, k-1$ 重复：
   - $R' = \text{PostOpt}(R)$ — 后处理优化（生成替换规则和修订规则，择优保留）
   - $D_i = \text{NotCovered}(R', D)$ — 获取未被覆盖的样例
   - $R_i = \text{IREP}^*(D_i)$ — 在剩余样例上学习新规则
   - $R = R' \cup R_i$ — 合并规则集

### FOIL算法 (西瓜书15.4)
1. 从空规则体开始（如"更好(X,Y) ←"）
2. 遍历所有候选文字（考虑变量组合），计算FOIL增益
3. 选择FOIL增益最大的文字加入规则体
4. 若规则仍覆盖反例，继续添加文字
5. 规则仅覆盖正例时，移去被覆盖正例，重复生成下一条规则
6. 后剪枝优化规则集

### LGG算法
1. 将正例转为具体事实规则（含所有背景知识文字）
2. 对两条初始规则找共同谓词文字
3. 逐位置比较常量：相同则保留，不同则引入新变量
4. 忽略仅有单条规则包含的谓词文字
5. 重复对多条规则求LGG，得到通用规则

## 直观理解 (Intuition)

**序贯覆盖就像剥洋葱**：每次剥掉一层（学一条规则、移去覆盖样本），剩下的是更小的核心问题。每次只需关注当前的"洋葱层"。

**规则的特化和泛化是"放大镜调焦"**：特化（自顶向下）是逐渐缩小覆盖范围，像调高放大倍数聚焦细节；泛化（自底向上）是放宽条件扩大视角，像调低放大倍数看全局。

**逆归结是"反向推理"**：逻辑演绎是从一般到特殊的"下山"（已知规则和条件推出结论），逆归结是从特殊到一般的"上山"（从具体例子反推一般规则），就像逆向工程一条定理的证明过程——找出需要什么前提才能推导出现有结论。

**LGG是"找最大公约数"**：两条规则就像两个数，LGG找的是能同时特化它们的"最大公因子"规则——既不能丢掉任何公有特征，又要恰好是最特殊的公共泛化。

## 常见误区 (Anti-patterns)

- **误区：规则越多越好**。过度生长的规则集会过拟合，需要剪枝和冲突消解。规则学习的优势恰在于简洁。

- **误区：一阶规则学习可以直接套用命题规则学习的方法**。逻辑变量的存在使得候选文字组合呈组合爆炸（如FOIL中需要测试变量X, Y, Z的各种搭配），需要特殊机制。

- **误区：LGG一定存在且唯一**。LGG对两条公式唯一存在证明成立，但需要正确的初始规则选择（RLGG通过引入背景知识 $K$ 来解决此问题）。

- **误区：ILP学出的新谓词语义明确**。逆归结可自动发明新谓词（如"q(M,N)"），但其语义完全由使用场景定义，只有领域专家才能赋予其实际含义。

```python
# 序贯覆盖基本框架
def sequential_covering(D_pos, D_neg, max_rules=None):
    rules = []
    while len(D_pos) > 0:
        rule = learn_single_rule(D_pos, D_neg)  # 自顶向下搜索
        if rule is None:
            break
        covered = get_covered(rule, D_pos)
        D_pos = [x for x in D_pos if x not in covered]
        rules.append(rule)
        if max_rules and len(rules) >= max_rules:
            break
    # 添加默认规则
    rules.append(DefaultRule(majority_class))
    return rules

# FOIL增益计算（只考虑正例信息量）
import math
def foil_gain(m_plus, m_minus, m_plus_new, m_minus_new):
    """原规则覆盖(m_plus, m_minus), 加入新文字后覆盖(m_plus_new, m_minus_new)"""
    info_old = math.log2(m_plus / (m_plus + m_minus)) if m_plus > 0 else 0
    info_new = math.log2(m_plus_new / (m_plus_new + m_minus_new)) if m_plus_new > 0 else 0
    return m_plus_new * (info_new - info_old)
```

关键实现注意：
1. 规则评估函数的设计（准确率、覆盖度、FOIL增益等）直接影响学得规则的质量
2. 集束搜索的束宽 $b$ 需权衡效率与全局最优性
3. 一阶规则学习中变量绑定的回溯机制是性能瓶颈

## 实例分析 (Worked Example)

**西瓜数据集2.0上的自顶向下规则学习**（西瓜书图15.1）：

初始空规则"好瓜←"覆盖所有样例。逐属性评估加入文字的准确率：
- "色泽=乌黑": 3/4，覆盖{2,3,7,15}（3正1反）
- "脐部=凹陷": 3/4

选"色泽=乌黑"加入规则。第二轮评估（在当前覆盖集上）：
- "根蒂=蜷缩": 100%，覆盖3个正例{2,3,7}

得到规则：`好瓜 ← (色泽=乌黑) ∧ (根蒂=蜷缩)`

使用集束搜索 ($b=2$)，第一轮保留两个最优文字，第二轮可获得规则：`好瓜 ← (色泽=乌黑) ∧ (纹理=稍糊)`，覆盖3个正例。

**西瓜数据集5.0上的LGG**（西瓜书）：

正例"更好(1,10)"和"更好(1,15)"的规则分别为：
- 更好(1,10) ← 根蒂更蜷(1,10) ∧ 声音更沉(1,10) ∧ 脐部更凹(1,10) ∧ 触感更硬(1,10)
- 更好(1,15) ← 根蒂更蜷(1,15) ∧ 脐部更凹(1,15) ∧ 触感更硬(1,15)

LGG将"10"与"15"替换为Y（忽略仅单条规则有的"声音更沉"），得：
`更好(1, Y) ← 根蒂更蜷(1, Y) ∧ 脐部更凹(1, Y) ∧ 触感更硬(1, Y)` (15.4)

再与瓜2的规则(15.5)求LGG，将常量进一步泛化为变量，得到不含常量的通用规则。

## 关键结论 (Key Takeaways)

1. **规则学习的核心优势是可解释性**：在需要理解模型决策逻辑的应用（如医疗诊断、法律推理）中不可替代
2. **序贯覆盖是所有规则学习算法的基石**：从AQ到FOIL到RIPPER，皆遵循"逐条学习→移去覆盖→重复"模式
3. **RIPPER是命题规则学习的巅峰**：其泛化性能首次超越决策树，核心在于IREP剪枝 + 后处理全局优化的组合
4. **FOIL是命题到一阶的桥梁**：通过变量替换将命题规则学习直接升维到一阶，效率高于ILP但表达能力受限
5. **逆归结让机器能"发明新概念"**：自动创造新谓词的能力是ILP区别于其他机器学习的独特之处
6. **规则学习与统计学习的融合是大趋势**：概率归纳逻辑程序设计、马尔可夫逻辑网等统计关系学习方法是前沿方向
