## 一、核心概念

**决策树**是一种以树形结构进行决策的机器学习模型。从根节点到叶节点的每条路径对应一条判定规则，叶节点给出最终预测结果。决策树可以视为 if-then 规则的集合，也可以视为定义在特征空间与类空间上的条件概率分布。

决策树学习的本质是从训练数据中归纳出一组分类规则，其关键挑战在于：
1. **如何选择划分属性**？（信息增益 / 增益率 / 基尼指数）
2. **如何处理连续值**？（二分法离散化）
3. **如何处理缺失值**？（概率权重分配）
4. **如何防止过拟合**？（预剪枝 / 后剪枝）

决策树模型的表达能力极强：在理论上，它可以逼近任意复杂的决策边界（只需将叶节点对应到训练样本本身即可实现训练集零误差）。

## 二、融合框架

### 决策树生成——统一递归框架

决策树的生成采用递归的分治策略：

```
函数 TreeGenerate(D, A):
    生成节点 node
    if D 中样本全属同一类别 C:
        将 node 标记为 C 类叶节点; return
    if A = ∅ OR D 中样本在 A 上取值相同:
        将 node 标记为 D 中样本数最多的类别; return
    从 A 中选择最优划分属性 a_*
    for a_* 的每个取值 a_*^v:
        为 node 生成一个分支; 令 D_v 表示 D 中在 a_* 上取值为 a_*^v 的样本子集
        if D_v = ∅:
            将分支节点标记为 D 中样本数最多的类别
        else:
            以 TreeGenerate(D_v, A \ {a_*}) 为分支节点
```

三种经典算法的区别在于**最优划分属性的选择标准**：

| 算法 | 选择标准 | 适用任务 | 树结构 | 年份 |
|------|----------|----------|--------|------|
| **ID3** | 信息增益 | 分类 | 多叉树 | 1986 |
| **C4.5** | 增益率 | 分类 | 多叉树 | 1993 |
| **CART** | 基尼指数（分类）/ 平方误差（回归） | 分类+回归 | 二叉树 | 1984 |

## 三、理论推导

### 3.1 信息增益（ID3）

设样本集合 $D$ 中第 $k$ 类样本的比例为 $p_k$，$D$ 的信息熵定义为：

$$H(D) = -\sum_{k=1}^{|\mathcal{Y}|} p_k \log_2 p_k$$

信息熵的取值下界为 0（当所有样本属于同一类时），上界为 $\log_2 |\mathcal{Y}|$（当各类均匀分布时）。

**证明上界**：由吉布斯不等式（或拉格朗日乘子法），在约束 $\sum p_k = 1$ 下最大化 $H = -\sum p_k \ln p_k$，解得 $p_k = \frac{1}{|\mathcal{Y}|}$，$H_{\max} = \ln |\mathcal{Y}|$。

用属性 $a$ 对 $D$ 划分后的**条件熵**：

$$H(D \mid a) = \sum_{v=1}^{V} \frac{|D^v|}{|D|} H(D^v)$$

**信息增益**：$Gain(D, a) = H(D) - H(D \mid a)$

信息增益越大，表示用属性 $a$ 划分后纯度提升越大。ID3 算法选择信息增益最大的属性进行划分。

**问题**：信息增益偏好取值数目多的属性。极端情况下，若每个样本有一个唯一 ID 属性，则信息增益最大但完全无法泛化。

### 3.2 增益率（C4.5）

为解决信息增益对取值数目多属性的偏好，C4.5 引入增益率：

$$Gain\_ratio(D, a) = \frac{Gain(D, a)}{IV(a)}$$

其中 $IV(a) = -\sum_{v=1}^{V} \frac{|D^v|}{|D|} \log_2 \frac{|D^v|}{|D|}$ 称为属性 $a$ 的固有值（Intrinsic Value）。$IV(a)$ 衡量属性 $a$ 的"分裂信息"——取值越多，$IV(a)$ 越大，增益率被压低。

**启示**：增益率偏好取值数目少的属性。C4.5 的实用策略是：先从候选属性中选出信息增益高于平均水平的属性，再从中选择增益率最高的属性。

### 3.3 基尼指数（CART）

样本集合 $D$ 的基尼值：$Gini(D) = \sum_{k=1}^{|\mathcal{Y}|} \sum_{k' \neq k} p_k p_{k'} = 1 - \sum_{k=1}^{|\mathcal{Y}|} p_k^2$

基尼值反映从 $D$ 中随机抽取两个样本类别标记不一致的概率——值越小纯度越高。

属性 $a$ 的基尼指数（在划分后）：$Gini\_index(D, a) = \sum_{v=1}^{V} \frac{|D^v|}{|D|} Gini(D^v)$

CART 分类树选择基尼指数最小的属性进行划分。

**基尼指数与信息熵的关系**：将 $-\ln p_k$ 在 $p_k = 1$ 处一阶泰勒展开：$-\ln p_k \approx 1 - p_k$，代入熵公式得 $H(D) \approx -\sum p_k \ln p_k \approx 1 - \sum p_k^2 = Gini(D)$。因此基尼指数是信息熵的一阶近似，使用基尼指数实际上是用简化版的信息熵衡量纯度。

### 3.4 CART 回归树

对于回归任务，CART 使用**平方误差最小化**选择最优切分。

设切分变量为第 $j$ 个特征，切分点为 $s$，定义两个区域：

$$R_1(j, s) = \{x \mid x^{(j)} \leqslant s\}, \quad R_2(j, s) = \{x \mid x^{(j)} > s\}$$

寻找最优 $(j, s)$ 使得：

$$\min_{j, s} \left[ \min_{c_1} \sum_{x_i \in R_1} (y_i - c_1)^2 + \min_{c_2} \sum_{x_i \in R_2} (y_i - c_2)^2 \right]$$

其中 $c_1, c_2$ 为各区域样本输出的均值。递归划分直到满足停止条件。

## 四、算法步骤

### 4.1 连续值处理——二分法

1. 对属性 $a$ 的取值排序：$a^1 \leqslant a^2 \leqslant \cdots \leqslant a^m$
2. 生成候选划分点集合 $T_a = \left\{\frac{a^i + a^{i+1}}{2} \mid 1 \leqslant i \leqslant m-1 \right\}$
3. 对每个 $t \in T_a$，按 $a \leqslant t$ 和 $a > t$ 二分计算信息增益
4. 选择信息增益最大的 $t$ 作为该属性的划分点

与离散属性不同的是：若当前节点划分属性为连续属性，该属性仍可作为其后代节点的划分候选属性（C4.5 设计）。

### 4.2 缺失值处理

C4.5 的缺失值处理策略（核心：按概率权重分配样本）：

给定属性 $a$，$D$ 中在 $a$ 上无缺失的样本子集为 $\tilde{D}$。设 $\tilde{D}^v$ 为 $\tilde{D}$ 中在 $a$ 上取值为 $v$ 的样本子集，$\tilde{D}_k$ 为 $\tilde{D}$ 中属于第 $k$ 类的样本子集。

为每个样本 $\mathbf{x}$ 赋予权重 $w_\mathbf{x}$（初值均为 1）：

- **信息增益公式推广**：$Gain(D, a) = \rho \times Gain(\tilde{D}, a)$

其中 $\rho = \frac{\sum_{\mathbf{x} \in \tilde{D}} w_\mathbf{x}}{\sum_{\mathbf{x} \in D} w_\mathbf{x}}$ 为无缺失样本比例。

- **样本划分策略**：若 $\mathbf{x}$ 在 $a$ 上取值缺失，将其以不同权重同时划入所有子节点：
  - 划入 $\tilde{D}^v$ 的权重为 $w_\mathbf{x} \cdot \frac{|\tilde{D}^v|}{|\tilde{D}|}$（按各取值比例分配）

### 4.3 剪枝

**预剪枝（Pre-pruning）**：在划分前用验证集评估划分收益，若泛化性能不提升则不划分。优点：训练快、开销小；缺点：可能因贪心策略欠拟合（当前划分虽无提升但后续划分可能有益）。

**后剪枝（Post-pruning）**：先生成完整决策树，再自底向上评估将内部节点替换为叶节点是否提升验证集性能。优点：通常比预剪枝保留更多分支，欠拟合风险小；缺点：训练开销大。

**CART 剪枝——代价复杂度剪枝**：剪枝后的损失函数为：

$$C_\alpha(T) = C(T) + \alpha |T|$$

其中 $C(T)$ 为训练误差（分类用基尼指数，回归用平方误差），$|T|$ 为叶节点数，$\alpha$ 控制复杂度惩罚强度。

CART 剪枝的核心技巧：生成从 $\alpha=0$（不剪枝）到 $\alpha=\infty$（剪到只剩根节点）的一系列子树序列 $\{T_0, T_1, \ldots, T_n\}$，然后用交叉验证选择最优子树。对每个内部节点 $t$，计算 $g(t) = \frac{C(t) - C(T_t)}{|T_t| - 1}$，每次剪去 $g(t)$ 最小的节点对应的子树。

## 五、直观理解

**为什么信息增益会偏好取值多的属性？** 想象一个极端的"学号"属性——每个样本的学号都不同。按学号划分后，每个子节点的信息熵为零（纯度最高），信息增益当然也最大。但这样的划分毫无泛化能力。增益率通过除以属性的"固有信息量"来惩罚取值过多的属性。

**基尼指数 vs 信息熵**：两者都能衡量集合纯度。基尼指数计算更简单（无对数运算），在 CART 中被采用。由于基尼值是熵的一阶近似，两者的决策树在多数情况下相似，但基尼指数倾向于产生更平衡的划分。

**预剪枝 vs 后剪枝的折中**：预剪枝像"三思而后行"——每次想分叉都先考虑是否值得；后剪枝像"撞了南墙再回头"——先放心大胆地构建完整树，再逐层回退多余分支。在实际应用中，若不考虑计算效率，后剪枝通常更好。

**连续值二分法的代价**：将连续值离散化为二分问题，降低了计算复杂度（从 $O(m \cdot V)$ 降至 $O(m \log m)$），但可能会丢失一些精细的划分信息。这是计算效率与模型精度之间的权衡。

## 六、常见误区

| 误区 | 正确理解 |
|------|----------|
| "决策树一定是二叉树" | ID3 和 C4.5 产生多叉树；只有 CART 强制每个节点二分 |
| "信息增益最大的属性就是最好的" | 信息增益偏好取值多的属性；C4.5 的增益率是更合理的标准 |
| "CART 只能用基尼指数" | 分类树用基尼指数，回归树用平方误差最小化 |
| "决策树很好理解所以一定不如复杂模型" | 在表格型数据上，梯度提升决策树（GBDT/XGBoost）仍是 SOTA |
| "剪枝后的树训练误差一定变大" | 剪枝会增大训练误差，目标是降低泛化误差 |

## 八、实例分析

**西瓜数据集 2.0** 是周志华《机器学习》中著名的教学示例，共 17 个样本、6 个离散属性。以此为例：

- ID3 选择"纹理"为根节点（信息增益 0.381 最高），产生三个分支（清晰、稍糊、模糊）
- C4.5 先选出信息增益高于平均水平的属性（纹理、根蒂、脐部、触感），其中"纹理"增益率也为最高
- 预剪枝在仅划分"脐部"后停止（验证精度不再提升），后剪枝保留更多分支（验证精度为 0.714 > 预剪枝的 0.571）

这个例子清晰展示了三种算法在属性选择上的差异，以及预剪枝/后剪枝对泛化性能的影响。

**现实场景——信用卡审批**：特征包含年龄（连续值）、收入（连续值）、是否有房产（离散值）、婚姻状况（离散值）、信用历史（可能缺失）。

- C4.5 自动对年龄和收入进行二分处理，对缺失的信用历史按比例分配权重
- 生成规则如："收入 <= 5000 且 无房产" --> 拒绝；"收入 > 5000 且 有房产" --> 批准
- 每条规则可被业务人员理解和审核——这是决策树相比黑箱模型的核心优势

## 九、关键结论

1. **三种分裂标准的关系**：信息增益（ID3）、增益率（C4.5）、基尼指数（CART，≈ 熵的一阶近似）
2. **剪枝是防止过拟合的关键**：预剪枝计算快但可能欠拟合，后剪枝泛化更好但计算开销大
3. **CART 的代价复杂度剪枝**提供了连续的剪枝调节——通过调节 $\alpha$ 在模型复杂度与精度间平滑过渡
4. **缺失值处理**的核心智慧：不丢弃数据，而是"按比例分配"——让样本以部分权重参与各分支
5. **决策树的本质**：通过递归划分将复杂的全局决策边界分解为一系列简单的局部规则——分而治之思想在机器学习中的完美体现
